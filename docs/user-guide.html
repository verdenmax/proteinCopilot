🧬 ProteinCopilot

AI 驱动的蛋白质组学质谱智能搜索平台

ProteinCopilot 是一个基于 Rust + MCP 协议 的蛋白质组学分析平台。用户通过自然语言与 AI 对话，AI 自动编排 19 个 MCP 工具完成从谱图读取到结果解读的全流程。

19MCP 工具
11Rust Crates
6内置数据库
5搜索预设
250+单元测试
0Clippy 警告

🎯 核心设计理念

Rust 负责确定性计算（谱图解析、搜索执行、FDR 计算），LLM 负责智能编排（意图理解、参数推荐、结果解释）。两者通过 MCP 协议严格分层，绝不混淆。

🚀 快速开始

1. 启动 MCP Server

在项目根目录放置 .mcp.json 配置：

{
  "mcpServers": {
    "protein-copilot": {
      "command": "cargo",
      "args": ["run", "--release", "-p", "protein-copilot-mcp-server"],
      "env": { "RUST_LOG": "info" }
    }
  }
}

Copilot CLI 或 Claude Desktop 会自动发现并启动 MCP Server。

2. 用自然语言开始分析

直接告诉 AI 你要做什么：

"帮我搜索 /data/sample.mzML，使用人的 Swiss-Prot 数据库"

"读取这个 mgf 文件，看看数据质量怎么样"

"有哪些可用的蛋白质数据库？下载小鼠的"

"对 scan 1234 做谱图注释"

3. AI 自动编排

AI 会自动调用对应工具，你只需确认关键参数：

1读取谱图read_spectra
→
2推荐参数recommend_params
→
3确认参数👤 用户确认
→
4执行搜索run_search
→
5监控进度get_search_status
→
6解读结果generate_summary
→
7导出文件export_results

🔄 搜索工作流

标准 DDA 搜索流程

用户: "帮我搜索 sample.mzML，人类样本"

AI 执行:
  1. read_spectra("sample.mzML")        → 获取谱图统计
  2. list_databases()                    → 检查 human_swissprot 缓存
  3. download_database("human_swissprot") → 下载(如未缓存)
  4. recommend_params(summary, hints)    → 推荐搜索参数
  5. [请用户确认参数]
  6. run_search(params, files, db_path)  → 启动异步搜索
  7. get_search_status(run_id)           → 轮询进度(每5-10秒)
  8. generate_summary(run_id)            → 生成 FDR 过滤后摘要
  9. [向用户解释结果]
  10. export_results(run_id, "./output") → 导出 TSV/JSON

DIA 数据流程

AI 执行:
  1. read_spectra("dia_sample.mzML")             → 检测 DIA 模式
  2. extract_dia_precursors("dia_sample.mzML")   → MS1 前体提取
  3. run_search(params, files, dia_run_id=...)    → 使用提取结果搜索
  4. generate_summary(run_id)                     → 结果摘要

外部结果导入 + 注释

用户: "导入 DIA-NN 的搜索结果，然后注释 scan 1234"

AI 执行:
  1. import_search_results(result.parquet, mzml_dir) → 导入 + 匹配 scan
  2. annotate_spectrum(run_id, scan=1234)             → 生成注释 HTML
  3. extract_xic(run_id, scan=1234)                   → 生成 XIC HTML

使用方式

用户: "有哪些可用的数据库？"        → list_databases

⚡ 搜索参数预设

Standard

通用

Trypsin · Carbamidomethyl(C) · Oxidation(M) · 10ppm/20ppm

适用：常规蛋白鉴定、HeLa、Shotgun

Phospho

磷酸化

Standard + Phospho(STY) 可变修饰

适用：磷酸化富集、信号通路

TMT

标记定量

TMT6plex(K, N-term) 固定修饰

适用：TMT 多路复用定量

SILAC

代谢标记

Heavy K(+8.014) R(+10.008) 可变修饰

适用：SILAC 定量、蛋白周转

Open Search

开放搜索

500 Da 前体偏差 · 发现未知修饰

适用：PTM 发现、质量偏移分析

🔬 谱图注释

annotate_spectrum 生成交互式 HTML，显示 b/y 离子匹配。支持两种模式：

模式 1：基于搜索结果

annotate_spectrum(
  run_id="xxx",
  scan_number=1234
)

自动从搜索结果获取肽序列、电荷、修饰

模式 2：手动指定

annotate_spectrum(
  file_path="data.mzML",
  scan_number=1234,
  peptide_sequence="PEPTIDE",
  charge=2
)

无需搜索结果，直接注释

📊 XIC 色谱提取

extract_xic 提取 MS1 前体和 MS2 碎片离子的色谱图，生成 Plotly.js 交互式 HTML。

支持 DIA 多 cycle 提取和 SILAC 轻/重标比较。

🔄 DIA 数据支持

ProteinCopilot 自动检测 DIA 数据（隔离窗口宽度 > 5 Da），提供完整的 DIA 分析流程：

  1. read_spectra — 自动检测 DIA 模式
  2. extract_dia_precursors — MS1 同位素模式提取前体候选
  3. run_search(dia_run_id=...) — 使用提取的伪谱搜索
  4. 或 import_search_results — 导入 DIA-NN Parquet 结果

🏷️ SILAC 分析

⚠️ 关键规则

SILAC 实验中，annotate_spectrum 和 extract_xic 必须传入 label_type 参数。遗漏会导致 DIA 窗口匹配错误。

四种场景矩阵

① DDA + 无 SILAC

标准注释谱图 + 标准 XIC

② DDA + SILAC

🪞 Mirror Plot — 轻/重标镜像对比

必须传 label_type: {"Silac": {...}}

③ DIA + 无 SILAC

标准注释 + DIA cycle XIC

④ DIA + SILAC

🪞 Mirror Plot + 双窗口 XIC

必须传 label_type: {"Silac": {...}}

SILAC label_type 示例

{
  "Silac": {
    "heavy_k_delta": 8.014199,   // ¹³C₆¹⁵N₂-Lys
    "heavy_r_delta": 10.008269   // ¹³C₆¹⁵N₄-Arg
  }
}

⚙️ 配置说明

{
  "mcpServers": {
    "protein-copilot": {
      "command": "cargo",
      "args": ["run", "--release", "-p", "protein-copilot-mcp-server"],

搜索引擎

MVP 阶段使用内置 SimpleSearchEngine（b/y 离子匹配），后续将接入 pFind 作为生产级搜索引擎。
