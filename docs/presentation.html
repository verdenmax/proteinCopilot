Model Context Protocol —— Anthropic 设计的开放协议
让 LLM 能够调用外部工具，就像浏览器能打开网页一样

📋 Tools

MCP Server 向 LLM 暴露"可调用的函数"。 每个 Tool 有名称、描述、JSON Schema 输入输出。 LLM 读描述后自主决定何时调用哪个 Tool。

📡 传输层

基于 JSON-RPC 2.0 协议。 支持 stdio（本地进程）和 SSE（远程 HTTP）两种传输。 ProteinCopilot 使用 stdio —— MCP Client 直接启动 Rust 进程。

🔌 即插即用

写好 MCP Server 后，无需修改即可接入任何 MCP Client。 Claude Desktop、Copilot CLI、Cursor 都能直接调用。 一次开发，多客户端受益。

类比：MCP 之于 LLM ≈ HTTP 之于浏览器

浏览器通过 HTTP 访问网页；LLM 通过 MCP 调用工具。协议标准化，生态开放。

03

🏗️ 核心设计：严格分层

系统分为两个世界：确定性 和 推理性，绝不混合

🦀 Rust 做什么（确定性）

✅ 谱图解析（mzML / mgf 二进制解码）

✅ 蛋白数据库酶切（Trypsin / Lys-C / ...）

✅ b/y 碎片离子理论值计算

✅ 谱图 vs 理论谱匹配打分

✅ FDR 统计（target-decoy · PSM/肽段/蛋白 三级）

✅ 蛋白推断（parsimony · razor peptide）

✅ XIC 提取 · DIA 前体检测

✅ 搜索诊断异常规则（7 条硬编码规则）

❌ 绝不调用 LLM API
❌ 绝不做"猜测"或"推理"

🤖 LLM 做什么（推理性）

✅ 理解用户自然语言意图

✅ 根据谱图特征推荐搜索参数

✅ 用通俗语言解释搜索结果

✅ 读取诊断报告 → 生成修复建议

✅ 规划多步骤工作流顺序

✅ 处理模糊请求（"帮我搜一下这个样本"）

✅ 失败时提供人类可读的诊断

✅ 跨工具协调（搜索 → 推断 → 可视化）

❌ 绝不做数值计算（FDR / 打分 / 质量）
❌ 绝不直接操作文件或进程

Rust — 可验证、可复现、有测试
LLM — 灵活、上下文感知、可解释
MCP — 连接两者的标准协议

🤖 LLM 如何串联整个流程

一次真实的蛋白质搜索对话：用户说了一句话，LLM 自动编排 6 个 MCP Tool 调用

👤 用户
→
"帮我搜一下 sample.mgf，用人的数据库"
🤖 LLM
→
意图理解：用户要做 DDA 搜索，物种=human，需要先读谱图 → 下载数据库 → 推荐参数 → 执行搜索
🤖 LLM
→
调用 read_spectra("sample.mgf") → Rust 返回：1,234 张 MS2 谱图，m/z 范围 300-1800
🤖 LLM
→
调用 download_database("human") → Rust 返回：UniProt human FASTA, 20,442 蛋白
🤖 LLM
→
调用 recommend_params(spectra_summary) → Rust 返回：Trypsin, 10ppm, b/y ions, Carbamidomethyl[C]
🤖 LLM
→
推理："谱图的碎裂模式和 m/z 分布符合 Trypsin 消化特征，推荐参数合理，直接启动搜索"
🤖 LLM
→
调用 run_search(params, database, spectra) → Rust 启动 Sage 引擎异步搜索...
🤖 LLM
→
解释结果："搜索完成！在 1% FDR 下鉴定到 856 个 PSM、623 个肽段、312 个蛋白质。 鉴定率 69%，属于正常水平。是否需要我进一步分析或导出结果？"

关键洞察：每一步数值计算都由 Rust 完成，LLM 只负责决定"下一步做什么"和"怎么解释"

🦀 Rust 实现逻辑详解

📂

谱图解析 (spectrum-io)

mzML XML + base64 解码 → f64 数组
MGF 文本流式解析
DDA/DIA 自动检测（IsolationWindow 宽度）
indexed 模式：按 scan 号随机访问
1 crate · 纯计算 · 无 I/O 副作用

🔬

搜索引擎 (search-engine)

SimpleSearch：b/y 碎片离子理论值 → 逐谱匹配打分
Sage：sage-core v0.15.0 库调用，fragment indexing + rayon 并行
Adapter trait 抽象 → 引擎可替换
打分公式、匹配逻辑全在 Rust 中

📊

FDR 控制 (fdr)

target-decoy 竞争策略
PSM → 肽段 → 蛋白 三级过滤
Picked-protein FDR（蛋白级别）
统计计算 100% 在 Rust 中，绝不交给 LLM

🧬

蛋白推断 (protein-inference)

Parsimony：最小蛋白集合覆盖所有肽段
Razor peptide：共享肽段分配给最大蛋白
蛋白分组 + 子集/超集关系
图算法 + 贪心算法，全部确定性

📈

XIC + DIA (xic · dia-extraction)

XIC 色谱图提取（m/z ± tolerance 窗口）
SILAC 轻/重标记 mirror plot
DIA：同位素模式检测 + MS1↔MS2 关联
数值插值 + 峰检测 = 纯数学

🔍

搜索诊断 (core/diagnostics)

7 条硬编码异常检测规则
阶段计时（fasta_parsing → matching → fdr）
错误分类（InputData/Parameters/Database/Engine）
规则是确定性的；LLM 负责解读

🤖 LLM 的角色：Agent + Skill 编排

LLM 不写代码、不做计算 —— 它是指挥官，通过 MCP 调度 Rust 工具

🎯

Agent 指令

.github/agents/proteomics-search.agent.md
定义 LLM 的"角色"：蛋白质搜索助手
列出 25 个可用工具 + 使用场景
LLM 读到这个指令后，知道自己有哪些能力

📋

12 个 Skill Prompts

预定义的分析"剧本"模板：
基础搜索 · 磷酸化搜索 · 开放搜索
DIA 分析 · 多引擎比较 · 失败诊断
LLM 按剧本调用 Tool 序列

🧠

推理决策

① 理解："帮我搜一下" → DDA 基础搜索
② 判断：谱图特征 → 适合 Trypsin 消化
③ 解释：FDR 1% 下 856 PSM = 正常水平
④ 诊断：ID 率 5% → 可能是数据库不匹配
每个判断都基于 Rust Tool 返回的数据

为什么 LLM 不直接算？

❌ 如果让 LLM 算 FDR：

"大约 1.2%... 或者可能是 0.8%..."
结果不确定、不可复现、无法审计

✅ 让 Rust 算 FDR：

q_value = 0.00847（精确到小数点后 5 位）
有单元测试、可复现、可审计

🔍 搜索诊断：Rust 规则 + LLM 解读

诊断功能完美体现了 Rust/LLM 分工：Rust 检测异常，LLM 解释原因

🦀 Rust 做的（确定性规则）

📏 记录每个阶段耗时（fasta → digestion → matching → fdr）

📊 计算指标：ID 率、FDR、decoy 比例

⚠️ 检测 7 类异常：

低 ID 率(<5%) · 高 FDR(>5%) · 零 decoy · 窄容差(<3ppm) · 宽容差(>50ppm) · 数据库不匹配 · 搜索过慢

🏷️ 分类错误：InputData / Parameters / Database / Engine

📋 输出结构化 JSON（anomalies + suggestions + stages）

🤖 LLM 做的（推理解释）

📖 读取 Rust 返回的诊断 JSON

🧠 综合判断根本原因：

"ID 率只有 3%，同时 decoy 命中为 0 → 很可能数据库物种不匹配"

💡 生成人类可读建议：

"建议检查样本物种，当前使用的是 human 数据库，尝试换用 mouse"

🔄 可以自动调整参数并重新搜索

📊 向用户解释每项异常的含义和影响

💡 为什么这样设计？

🔒

可复现性

科学计算必须可复现。
同样的输入 → Rust 永远给出同样的结果。
LLM 的输出有随机性，不适合做数值计算。
每次运行记录 run_id + 参数 + 版本

⚡

性能

质谱搜索需要处理百万级谱图。
Rust 零成本抽象 + rayon 并行。
Sage 引擎每秒处理数千张谱图。
LLM 调用延迟 >1s，不适合高频计算

🧪

可测试性

670+ 测试 · 0 clippy warnings。
每个 crate 独立编译和测试。
FDR 计算有精确的预期值断言。
LLM 输出无法写精确断言

🗣️

自然语言理解

"帮我搜一下" → DDA 基础搜索
"磷酸化分析" → 添加 STY 修饰
这种模糊指令只有 LLM 能理解。
传统工具需要用户精确配置 200+ 参数

📖

结果解释

"q-value = 0.00847" → 人类看不懂
LLM：该蛋白在 1% FDR 下高置信鉴定"
降低蛋白组学分析的专业门槛。
让非专家也能理解搜索结果

🔌

生态开放

MCP 是开放标准协议。
Claude / GPT / Gemini 都可以做 Client。
一次开发 Rust Server → 多 AI 受益。
不绑定任何特定 LLM 供应商
